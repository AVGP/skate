SkateJS

Skate is a component-based micro-framework meant to give you a common interface for interacting with existing and new DOM elements given only a selector.

Why?

Skate is heavily inspired by the SelectorListener technique by Daniel Buchner. Currently, the only way to accomplish handling elements added to the DOM is by using Mutation Observers which are inherently slow and are only available in the greenest of evergreen browsers. Contrary to Mutation Observers, this technique uses CSS animation keyframes to notify handlers about elements that are added to the DOM. This works for both new elements and existing elements alike and is very performant.

Creating a Component

Creating a new component out of existing and new DOM elements is as easy as calling a function and passing a CSS selector.

        skate('p', function(element) {
          ...
        });

Interacting with a Component

When you bind a component to a subset of elements, a component instance is returned. This allows you to interact with the elements bound to that component.

        var p = skate('p', function() {
          ...
        });

        // Shorthand for traversing each element the component controls that is
        // currently in the DOM.
        this.each(function(el) {
          ...
        });

        // [el1, el2, ...]
        p.elements();

        // Stop listening for new elements.
        p.destroy();

Customising a Component

In addition to being able to call methods on a component, you can also define your own methods.

        var p = skate('p[lang]', {
          init: function(element) {
            // The main setup method that gets called for each element.
          },
          translate: function(to) {
            this.each(function(element) {
              // Some pseudo-code for thought.
              element.innerHTML = translate(
                element.innerHTML,
                element.getAttribute('lang'),
                to
              );
            });
          }
        });

        p.translate('en');

Compatibility

Works in all evergreen browsers that support CSS animation and falls back to using deprecated mutation events in IE9.
